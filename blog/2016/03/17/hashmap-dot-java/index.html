HashMap源码分析

Mar 17th, 2016 7:37 pm

HashMap在Map框架图中的位置

1.定义

HashMap实现了Map接口，继承了AbstractMap类，其中Map接口定义了键值的映射规则，而AbstractMap类提供了Map接口的基本实现.

HashMap其中一个构造函数

由上面HashMap的构造函数可知,每次new一个HashMap对象,都会先初始化一个table数组,table数组的元素为Entry对象.

Entry为HashMap的静态内部类

Entry为HashMap的静态内部类,它的对象包括了键key、值value、下一个Entry结点next以及hash值.table数组下的链表就是由一个个Entry结点组成的.

计算添加元素的位置

存储元素——put函数

put函数源码

put函数原理:当我们想一个HashMap中添加一对key-value时，系统首先会计算key的hash值，然后根据hash值确认在table中存储的位置。 若该位置没有元素，则直接插入。否则迭代该处元素链表并依此比较其key的hash值。 如果两个hash值相等且key值相等(e.hash == hash && ((k = e.key) == key || key.equals(k))),则用新的Entry的value覆盖原来节点的value。如果两个hash值相等但key值不等 ，则将该节点插入该链表的链头

添加Entry结点——addEntry函数源码

addEntry中，对当前要添加的结点都是作为链表的头结点存储在table数组的下标bucketIndex处，对于该处已经有的结点或已经形成的结点链表则要连接在新加入的结点后面.

HashMap添加键为null的处理函数putForNullKey

对于键key为null的情况由putForNullKey函数专门处理
对于键为null的结点只可能存储在table[0]处或table[0]为链表头的链表上。如果table[0]的值为null，说明table[0]处还没有 存入结点。可以直接将该null键新建的结点存储在table[0]即可。 如果table[0]处的值不为null，说明该处已经存储有结点或结点形成的链表。table[0]处不止存储有键为null的结点，还可能有别的键不为null的结点. 如果先存入的是键为null的结点后存入的是键不为null的结点，那么键为null的结点可能在链表的中间，所有此时要遍历table[0]的链表。如果此链表中 有键为null的结点，更新该结点的处null键所对应的值即可。如果遍历完链表没有找到键为null的结点，那么就要将键为null的新结点作为链表头存储在 table[0]处了，原来的链表结点都加在新结点后面。

HashMap中可以存入键为null的键值对，但只能存入一个这样的键值对，而且位置是在table[0]处。
