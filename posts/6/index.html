HashMap在Map框架图中的位置

1.定义

HashMap实现了Map接口，继承了AbstractMap类，其中Map接口定义了键值的映射规则，而AbstractMap类提供了Map接口的基本实现.

2.HashMap类中变量

static final int DEFAULT_INITIAL_CAPACITY = 16;//默认的初始容量，必须是2的幂


static final int MAXIMUM_CAPACITY = 1 << 30;//最大容量（必须是2的幂且小于2的30次方，传入容量过大将被这个值替换）


static final float DEFAULT_LOAD_FACTOR = 0.75f;//负载因子  尺寸/容量

//HashMap的底层主要是基于数组和链表来实现,其中的数组指的的table数组
//Entry为HashMap类的内部类
transient Entry[] table;


transient int size;//存的键值对的数量


int threshold;//阈值 当实际大小超过阈值时，会进行扩容  阈值=容量*加载因子


final float loadFactor;//加载因子


transient int modCount;//map结构被改变的次数

HashMap中的数据结构

HashMap底层的数组加链表结构

HashMap其中一个构造函数

由上面HashMap的构造函数可知,每次new一个HashMap对象,都会先初始化一个table数组,table数组的元素为Entry对象.

Entry为HashMap的静态内部类

Entry为HashMap的静态内部类,它的对象包括了键key、值value、下一个Entry结点next以及hash值.table数组下的链表就是由一个个Entry结点组成的.

计算添加元素的位置

存储元素——put函数

put函数源码

put函数原理:当我们想一个HashMap中添加一对key-value时，系统首先会计算key的hash值，然后根据hash值确认在table中存储的位置。 若该位置没有元素，则直接插入新结点。否则迭代该处元素链表并依此比较其key的hash值。
如果两个hash值相等且key值相等(e.hash == hash && ((k = e.key) == key || key.equals(k))),则用新的Entry的value覆盖原来节点的value。
如果两个hash值相等但key值不等 ，则将该节点插入该链表的链头

添加Entry结点——addEntry函数源码

addEntry中，对当前要添加的结点都是作为链表的头结点存储在table数组的下标bucketIndex处，对于该处已经有的结点或已经形成的结点链表则要连接在新加入的结点后面.

HashMap添加键为null的处理函数putForNullKey

对于键key为null的情况由putForNullKey函数专门处理
对于键为null的结点只可能存储在table[0]处或table[0]为链表头的链表上。如果table[0]的值为null，说明table[0]处还没有 存入结点。可以直接将该null键新建的结点存储在table[0]即可。
如果table[0]处的值不为null，说明该处已经存储有结点或结点形成的链表。table[0]处不止存储有键为null的结点，还可能有别的键不为null的结点.
如果先存入的是键为null的结点后存入的是键不为null的结点，那么键为null的结点可能在链表的中间，所有此时要遍历table[0]的链表。如果此链表中 有键为null的结点，更新该结点的处null键所对应的值即可。如果遍历完链表没有找到键为null的结点，那么就要将键为null的新结点作为链表头存储在 table[0]处了，原来的链表结点都加在新结点后面。

HashMap中可以存入键为null的键值对，但只能存入一个这样的键值对，而且位置是在table[0]处。

删除元素——remove函数

给定键key,可以删除HashMap对象中的键值对

public V remove(Object key) 
{
    Entry<K,V> e = removeEntryForKey(key);
    return (e == null ? null : e.value);
}

final Entry<K,V> removeEntryForKey(Object key) 
{
    int hash = (key == null) ? 0 : hash(key.hashCode());

    int i = indexFor(hash, table.length);//计算其数组下标

    Entry<K,V> prev = table[i];

    Entry<K,V> e = prev;

    while (e != null) 
    {
        Entry<K,V> next = e.next;
        Object k;

        if (e.hash == hash && ((k = e.key) == key || (key != null && key.equals(k)))) 
        {
            modCount++;

            size--;

            if (prev == e)
            {
                table[i] = next;
            }
            else
            {
                prev.next = next;
            }

            e.recordRemoval(this);

            return e;
        }

        prev = e;
        e = next;
    }

    return e;
}

HashMap扩容

使用put函数向HashMap的对象中添加结点时，如果存储的键-值对个数大于等于阈值threshold时，会调用resize函数对当前HashMap对象进行扩容。一次扩容是将table数组的大小增长1倍.

void resize(int newCapacity)//hashmap扩容,重新hash
{
    Entry[] oldTable = table;

    int oldCapacity = oldTable.length;

    //如果旧的容量已经是MAXIMUM_CAPACITY,则传入新的大小之后也不会重新扩容
    //已经到可以支持的最大容量
    //只是将旧的阈值改为Integer.MAX_VALUE

    if (oldCapacity == MAXIMUM_CAPACITY)
     {
        //增大阈值，阈值太大可能造成冲突增多
        threshold = Integer.MAX_VALUE;

        return;
    }

    //新的table数组
    Entry[] newTable = new Entry[newCapacity];

    //将旧table数组的结点全部转移到新的table数组中
    transfer(newTable);

    table = newTable;

    threshold = (int)(newCapacity * loadFactor);
}

transfer函数将旧tables数组中的所有结点转移到新的table数组中.

//将旧数组中的entry转移到新数组中
void transfer(Entry[] newTable)
{
    Entry[] src = table;//旧table数组

    int newCapacity = newTable.length;

    for (int j = 0; j < src.length; j++) 
    {
        Entry<K,V> e = src[j];

        if (e != null) 
        {
            src[j] = null;

            //处理结点链表
            do 
            {
                Entry<K,V> next = e.next;

                //重新计算数组下标
                int i = indexFor(e.hash, newCapacity);

                e.next = newTable[i];

                newTable[i] = e;

                e = next;
            } while (e != null);
        }
    }
}

Java语言的"编译期"其实是一段"不确定"的操作过程,因为它可能指以下几个:
1. 前端编译器，把.java文件转变为.class文件的过程;

3. 使用静态提前编译器(AOT编译器)直接把*.java文件编译成本地机器代码的过程;

Java访问权限修饰词:

类的访问控制

类访问的含义是：若一个类(类A)访问另一个类(类B)时，其所指类A有以下3中操作:
1. 在类A中创建类B的对象;
2. 使用类B中某些方法和成员变量;
3. 继承类B;

类的访问权限有两种，分别是：
1. 公有级别访问(public)
2. 默认访问级别 (可看作包访问)

类的成员访问控制

  1. public：被public修饰的方法或者变量,所有其他类，无论该类属于那个包，均能访问该成员
  2. 默认访问权限(包访问权限): 对同一个包中的类，默认类型相当于public,而对于包外的类则相当于private
  3. protected 如果一个类的方法或者变量被protected修饰，对于同一个包的类，这个类的方法或变量是可以被访问的。对于不同包的类，只有继承于该类的类才可以访问到该类的方法或者变量
  4. private：如果一个类的方法或者变量被private修饰，那么这个类的方法或者变量只能在该类本身中被访问，及其不能被该成员所在类之外的任何类中的代码访问.

final关键字

用final修饰变量的基本含义是,改变量一旦被显示初始化之后其值就不能再被修改. 但对基本类型和引用类型要区别理解.

对于基本类型,final使其数值恒定不变
对于引用类型,final使改引用恒定不变，即一旦引用被初始化指向一个对象，就无法再把它改为指向另一个对象;但是,被指向对象本身却是可以修改的.

空白final
待写部分

final参数
Java允许在参数列表中以声明的方式将参数指明为final,这意味着无法再方法中更改参数引用所指向的对象.

final修饰方法
把方法锁定,以防止任何继承类修改它的含义.这确保了在继承中使方法行为保持不变,并且不会被覆盖(override).

final修饰类
final类禁止继承,所以final类中所有方法都隐式指定为final的，因为无法覆盖(override)它们.

static关键字

static关键字只能修饰成员.

  1. static方法

  2. static变量

  3. static代码块

java接口

java接口定义

<访问限制修饰符> [abstract] interface <接口名>
{

}

  1. 其中访问限制修饰符与类的访问限制修饰符相同,可以是public或不写(即默认包访问权限),
    使用public修饰的接口必须必须接口名是在与其同名的文件中(与public 类一样).
  2. [abstract]可选，可写可不写，如果不写，系统在进行编译时也会自动加上，也就是说接口一定是抽象的

接口不能有构造函数

因为接口声明中隐含了abstract，所以不能用final关键字来修饰接口.

java接口中的成员变量:

接口也可以包含域(成员变量)，但是这些域隐式地是static和final的
可以在接口中不必显示地使用public,static,final关键字，及时不使用这些修饰符, 编译时也会为其自动加上.
接口中的变量隐含是"public static final"的

java接口中的方法:

  1. 所有接口方法隐含都是公有的(public)和抽象的(abstract)，不需要显示声明;
  2. 接口方法一定不能是静态的(static)，也一定不能是final的；也不能用public修饰符之外的其他访问限制符进行修饰; 也就是说，要给接口中的方法写修饰符,只能写public和abstract

Java接口常用方法：

  1. 单方法接口
    一个单方法接口中只有一个方法
  2. 标识接口
    标识接口是没有任何方法和属性的接口.标识接口不对实现它的任何类有任何语义上的要求,它仅仅表明实现它的类属于一个特定的类型.
    标识接口在java语言中有一些很著名的应用,比如java.io.Serializable便是标识接口.
    java jdk7中java.io.Serializable接口源代码

3.常量接口
所谓常量接口，是指用Java接口来声明一些常量，然后由实现这些接口的类使用这些常量.
凡是实现了这个接口的Java类都会自动继承这些常量，并且都可以像使用自己的常量一样使用这些常量
不过实际使用中不建议这样使用常量接口
